Architecture

AlphaFold Sovereign MCP implements a hybrid local-first architecture:

  1. Primary: Check local PDB cache (dynamically indexed)
  2. Fallback: Fetch from AlphaFold DB (no API key required)
  3. Auto-cache: Fetched structures saved locally for sovereignty
  4. Compute: Extract features and topology on-demand

Mathematical Foundation

Built on rigorous topological data analysis:

  • Persistent homology via Vietoris-Rips filtration
  • Betti numbers (connected components, loops, voids)
  • Wasserstein and bottleneck distances for comparison
  • GO semantic similarity with information content

Installation

Requirements

  • Python 3.10+
  • numpy
  • pydantic
  • mcp (Model Context Protocol)

Install from Source

git clone https://github.com/smaniches/alphafold-sovereign-mcp.git
cd alphafold-sovereign-mcp
pip install -e .

Claude Desktop Integration

Add to your claude_desktop_config.json:

{
  "mcpServers": {
    "alphafold-sovereign": {
      "command": "python",
      "args": ["-m", "alphafold_sovereign"],
      "env": {
        "ALPHAFOLD_STRUCTURES_DIR": "/path/to/pdb_files",
        "ALPHAFOLD_CACHE_DIR": "/path/to/cache"
      }
    }
  }
}

Configuration

Configuration is loaded from multiple sources with the following priority:

PrioritySourceDescription
1 (highest)Environment variablesALPHAFOLD_STRUCTURES_DIR, ALPHAFOLD_CACHE_DIR, ALPHAFOLD_CACHE_MODE
2~/.alphafold_sovereign/config.jsonUser configuration file (recommended)
3~/.config/alphafold_sovereign/config.jsonXDG config location
4 (lowest)Module defaultsPlatform-specific defaults

Configuration Options

KeyTypeDescription
structures_dirstringPath to local AlphaFold PDB files directory
cache_dirstringPath to cache directory for annotations and metadata
cache_modestringsovereign | readonly | disabled

Core Tools

get_structure
CORE
Retrieve an AlphaFold protein structure by UniProt ID. Checks local cache first, falls back to AlphaFold DB online.
Parameters
uniprot_id requiredstringUniProt accession (e.g., 'P12345')
include_featuresbooleanCompute secondary structure, binding pockets (default: true)
include_topologybooleanCompute persistent homology (default: false)
force_onlinebooleanForce fetch from AlphaFold DB (default: false)
search_structures
CORE
Search local AlphaFold structure cache by UniProt ID pattern using glob-style matching.
Parameters
patternstringGlob-style pattern (e.g., 'A0A*', 'P123*') (default: '*')
limitintegerMaximum results (default: 100, max: 1000)
batch_structures
CORE
Retrieve multiple AlphaFold structures in a single request. Up to 50 proteins.
Parameters
uniprot_ids requiredarrayList of UniProt IDs (max 50)
include_featuresbooleanInclude structural features (default: true)
get_features
CORE
Compute detailed structural features: secondary structure, binding pockets, confidence metrics.
Parameters
uniprot_id requiredstringUniProt accession ID
feature_typesarrayFeatures: secondary_structure, binding_pockets, confidence, contacts
get_topology
CORE
Compute topological features via persistent homology on C-alpha atoms using Vietoris-Rips filtration.
Parameters
uniprot_id requiredstringUniProt accession ID
max_dimensionintegerMaximum homology dimension 0-2 (default: 2)
max_filtrationnumberMaximum filtration radius in Angstroms (default: 25)
check_availability
CORE
Check availability of structures in local cache and AlphaFold DB online.
Parameters
uniprot_ids requiredarrayList of UniProt IDs to check (max 100)
get_cache_statistics
CORE
Get statistics about the local structure cache: count, directories, mode, storage locations.
Parameters
(none)-No parameters required
